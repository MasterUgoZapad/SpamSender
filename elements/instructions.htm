Инструкция пользователя.

Пользователи системы.

Для работы с системой нужно быть зарегистрированным пользователем и войти в систему под своим логином и паролем.

Внимание, незалогиненные пользователи автоматически перенаправляются на страницу входа в систему.

Существует несколько уровней привилегий пользователей:

-        0 – незалогиненный пользователь. Может только логиниться и выходить из системы.

-        1 – обычный пользователь. Может совершать любые действия, связанные с непосредственным функционированием системы.

-        2 – администратор. Может совершать любые действия, в том числе управлять аккаунтами других пользователей и совершать массовый импорт данных.

Внимание, если у вас нет аккаунта в системе, попросите любого администратора создать его. Не существует других способов создания аккаунта.

При завершении работы с системой выйдите из нее.

Посылка писем

Чтобы послать письмо отдельному сохраненному человеку, перейдите сначала на страницу просмотра адресатов и выберите опцию «послать письмо»

Чтобы отправить письмо группе людей, перейдите на страницу группы и создайте новую. Затем, вы сможете на странице отправки писем выбрать эту группу или на странице просмотра групп выбрать опцию «послать письмо».

Чтобы оправить письмо на почтовый адрес, введите его в поле адреса.

При отправке письма могут быть применены шаблоны. Шаблоны могут содержать специальные конструкции, которые в готовом письме будут заменены соответствующими персональными данными адресатов.

Внимание, при отправке писем на простой почтовый адрес с применением шаблона, данные в шаблоне не будут заполнены.

Внимание, отправка писем является длительной операцией.

Группы.

Контакты могут быть объединены в группы для более удобной массовой рассылки писем.

На странице просмотра групп вы можете увидеть полный список сохраненных групп.

При клике по группе вы увидите полный список входящих в нее контактов.

Группы могут быть добавлены и удалены, можно послать письма отдельной группе.

Группа создается путем выборки контактов соответствующих задаваемому шаблону.

При задании шаблона строковые поля контактов могут быть заданы регулярными выражениями, состоящими из символов, символа «_», означающего один любой символ и символа «%», означающего любое количество любых символов.

Поля, имеющие формат даты - дата регистрации и дата рождения задаются верхней и нижней допустимой границей.

Внимание, перед сохранением данных о группе следует обновить и просмотреть список, потенциально входящих в нее контактов.

Шаблоны писем.

Для более удобной рассылки писем существует возможность сохранять шаблоны писем.

Шаблон включает в себя текст письма, тему письма и название шаблона.

На странице просмотра шаблонов писем  вы можете увидеть полный список сохраненных шаблонов.

При клике по шаблону вы увидите содержащийся в нем текст и тему письма.

Шаблоны могут быть добавлены, удалены и изменены, можно применить шаблон к отправляемым письмам.

Текст шаблона может содержать синтаксические конструкции, которые при посылке письма будут заменяться на данные конкретного контакта, которому будет отправлено письмо.

Внимание, замена конструкций данными происходит только при применении сохраненного шаблона. Если добавить эти конструкции в обычное письмо, замена не будет произведена.

Для удобства использования при создании нового шаблона демонстрируется список доступных конструкций.

Колонки с другими именами будут проигнорированы.

Так же можно задать значения по умолчанию для всех полей адресатов. Значения по умолчанию будут применяться в том случае, когда нужная колонка будет отсутствовать или ее значение будет пустым или не валидным.

Внимание, при загрузке файлов содержащих кириллические символы, убедитесь в верности выбранной кодировки.

Внимание, массовый импорт является длительной операцией.

Длительные операции.

Некоторые операции, такие как отправка писем и массовый импорт данных могут занимать ощутимое время при большом объеме данных.

Разрешен запуск только одной длительной операции единовременно. На выполнение длительной операции указывает специальный индикатор, показывающий, какой этап операции сейчас выполняется.

Внимание, попытка запуска длительной операции во время выполнения другой длительной операции приведет к ошибке, но не приведет к прерыванию уже запущенной операции.
